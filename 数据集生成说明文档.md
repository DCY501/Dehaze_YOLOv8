# VOC_hazy 多浓度数据集生成说明

> 面向队友：本文档说明如何生成"一对多"多浓度雾图数据集，供 baseline / AOD / Dehaze 等各类实验统一使用。

---

## 一、为什么改数据集？

原来的数据集是**一对一**：一张清晰图只合成一张固定 beta=1.35 的雾图。

**问题**：
- 模型只在单一雾浓度上训练，泛化性差
- 无法适应真实场景中不同浓度的雾天

**改进**：参考 DR-YOLO（Pattern Recognition 2024），一张清晰图合成 **5 种不同浓度**的雾图，全部放入训练集。模型在训练时就能见到从轻雾到浓雾的全谱分布，泛化能力显著提升。

---

## 二、5 种浓度选取

| beta | 雾浓度 | 说明 |
|:---:|:---|:---|
| 0.60 | 轻雾 | 接近清晰，保留大部分细节 |
| 0.80 | 轻雾 | 常规轻雾场景 |
| 1.00 | 中雾 | 适中难度 |
| 1.35 | 中雾 | **原数据集固定值**，作为对照锚点 |
| 1.80 | 浓雾 | 挑战性场景，增强鲁棒性 |

> 浓度覆盖区间：0.6 ~ 1.8，从"几乎无雾"到"明显浓雾"，不包含极端重雾（避免信息丢失过多）。

---

## 三、训练时间控制：30% 分层子集

**问题**：5 种浓度 × 2501 张 = 12,505 张 train 图，全量训练时间太长（~25 小时）。

**方案**：每种浓度 deterministic 抽取 **30%** 子集，合并为训练集。

| 项目 | 数值 |
|:---|:---|
| 单浓度图像数 | ~2,501 张 |
| 每浓度子集 | 750 张（30%） |
| 合并训练集 | **3,750 张** |
| 每 epoch 迭代数 | ~469 steps（batch=8） |
| 50 epoch 预估时间 | **~4~5 小时** |

**子集策略**：
- **分层抽样**：每种浓度内部独立随机抽 30%，不同浓度之间不互相影响
- **固定种子**：`seed=42`，保证每次生成结果完全一致，baseline 和 dehaze 实验绝对对齐
- **过滤空标签**：只保留包含至少一个有效标注目标的图像（排除 difficult=1 和无目标图像）

## 四、文件结构

生成后目录如下：

```
datasets/VOC_hazy_5beta/
├── images/
│   ├── train/          # 12,505 张雾图（5 beta × 2501）
│   ├── val/            # 12,550 张雾图（5 beta × 2510）
│   └── test/           # 24,760 张雾图（5 beta × 4952）
├── clean/
│   ├── train/          # 12,505 张清晰图（带 beta 后缀，一一对应）
│   ├── val/
│   └── test/
├── labels/
│   ├── train/          # 12,505 个标签（与雾图同名）
│   ├── val/
│   └── test/
├── train_5beta_subset30.txt      # ← 训练子集列表（3,750 张）
├── train_beta0.60_subset30.txt   # 单 beta 子集（调试用）
├── train_beta0.80_subset30.txt
├── train_beta1.00_subset30.txt
├── train_beta1.35_subset30.txt
├── train_beta1.80_subset30.txt
└── VOC_hazy.yaml                 # ← 数据集配置文件
```

**重要**：
- Val / Test **不做子集**，保持全量，用于稳定评估
- `clean/` 目录下的清晰图也带 `beta` 后缀（如 `000005_beta0.60.jpg`），确保与雾图一一匹配
- 子集 txt 里的路径格式为 `./images/train/xxx_betaX.jpg`，YOLO 能正确解析

## 六、训练时如何引用

### 基线 / AOD / Dehaze 通用

训练脚本中 `--data` 参数统一指向：

```bash
--data datasets/VOC_hazy_5beta/VOC_hazy.yaml
```

### 示例：基线训练

```bash
python scripts/train.py --mode baseline --epochs 50 \
  --data datasets/VOC_hazy_5beta/VOC_hazy.yaml \
  --batch 8 --workers 0 \
  --name baseline_5beta_subset30_50e
```

### 示例：多任务训练（Dehaze）

```bash
python scripts/train.py --mode dehaze --epochs 50 \
  --data datasets/VOC_hazy_5beta/VOC_hazy.yaml \
  --batch 8 --workers 0 --dehaze 0.05 \
  --name dehaze_5beta_subset30_50e
```

### 示例：断点续训

```bash
python scripts/train.py --mode baseline --epochs 50 \
  --data datasets/VOC_hazy_5beta/VOC_hazy.yaml \
  --batch 8 --workers 0 \
  --name baseline_5beta_subset30_50e \
  --resume
```

## 七、对照实验注意事项

1. **统一数据**：所有对比实验（baseline vs AOD vs Dehaze）必须使用**同一套** `train_5beta_subset30.txt`，确保数据对齐
2. **不要改 seed**：子集生成种子固定为 `42`，改 seed 会导致子集变化，对照不公平
3. **epoch 统一**：建议先统一跑 50 epoch，结果可信后再扩展到 100 epoch
4. **val / test 全量**：验证和测试始终用全量 5-beta 数据，不做子集，保证评估稳定性
